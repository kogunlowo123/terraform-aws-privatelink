Terraform ≥ 1.5.0AWS Provider ≥ 5.20.0MIT License

Executive Summary

This Terraform module provisions AWS PrivateLink infrastructure supporting both producer and consumer patterns. It enables private connectivity between VPCs and AWS services without exposing traffic to the public internet. The module manages VPC Endpoint Services backed by NLB/GWLB, VPC Endpoints (Interface and Gateway types), security groups, endpoint policies, allowed principals, and connection acceptance workflows.

Data Flow

  1. Producer deploys application behind a Network or Gateway Load Balancer
  2. VPC Endpoint Service is created, referencing the NLB/GWLB ARN(s)
  3. Allowed Principals are registered to restrict who can create consumer endpoints
  4. Consumer creates a VPC Endpoint (Interface type creates ENIs in specified subnets)
  5. Connection acceptance is handled automatically or manually per configuration
  6. Traffic flows over the AWS backbone network, never traversing the public internet
  7. Endpoint Policies and Security Groups enforce least-privilege access
  8. Private DNS enables transparent hostname resolution within the consumer VPC

Security Controls

Network Isolation

Traffic never traverses the public internet. All communication stays on the AWS backbone via PrivateLink.

Security Groups

Auto-created SG restricts inbound to HTTPS (port 443) from VPC CIDR blocks only.

Principal-Based Access

Allowed principals list restricts which AWS accounts/roles can discover and connect to the endpoint service.

Endpoint Policies

IAM policy documents attached to endpoints provide fine-grained resource and action-level access control.

Connection Acceptance

Manual or automatic acceptance workflow ensures no unauthorized connections are established.

Private DNS

Private DNS names resolve within the VPC, preventing accidental exposure through public DNS resolution.

Production Readiness Checklist

  • Enable acceptance_required for producer endpoint services
  • Define explicit allowed_principals (do not leave open)
  • Attach endpoint policies to restrict actions/resources
  • Use Interface endpoints with private_dns_enabled for seamless integration
  • Deploy Interface endpoints across multiple subnets/AZs
  • Review default security group rules and customize if needed
  • Tag all resources for cost allocation and governance
  • Use Gateway endpoints for S3 and DynamoDB (no additional cost)
  • Monitor VPC endpoint metrics via CloudWatch
  • Test failover behavior for multi-AZ endpoint deployments

Deployment

# Consumer - create endpoints for AWS services
module "vpc_endpoints" {
  source = "kogunlowo123/privatelink/aws"

  name   = "my-app"
  vpc_id = "vpc-0123456789abcdef0"

  endpoints = {
    s3 = {
      service_name = "com.amazonaws.us-east-1.s3"
      type         = "Gateway"
    }
    ssm = {
      service_name        = "com.amazonaws.us-east-1.ssm"
      type                = "Interface"
      subnet_ids          = ["subnet-aaa", "subnet-bbb"]
      private_dns_enabled = true
    }
  }

  tags = { Environment = "production" }
}

# Producer - expose a service via PrivateLink
module "endpoint_service" {
  source = "kogunlowo123/privatelink/aws"

  name   = "my-service"
  vpc_id = "vpc-producer123"

  create_endpoint_service    = true
  network_load_balancer_arns = [aws_lb.nlb.arn]
  acceptance_required        = true
  allowed_principals         = ["arn:aws:iam::111111111111:root"]
  create_vpc_endpoints       = false

  tags = { Environment = "production" }
}

terraform init
terraform plan
terraform apply
